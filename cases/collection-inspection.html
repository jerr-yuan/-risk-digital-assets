The Soul

系统的灵魂：编码的专家经验

"代码可以开源，但编码在系统里的战略判断力无法复制。这正是'资深策略专家的经验复制'—— 不是复制工具，是复制思维。"

📖

数据字典 & 指标体系

不是简单的字段说明，而是每个指标的业务含义、正常区间、异常信号判定规则。新人看懂指标，资深团队校准口径。

✅

归因 Checklist

三条监控线各有完整的归因检查清单。策略团队按此蓝本看数、分析、下结论。不依赖个人经验，而是系统性排查。

🧠

诊断 Business Sense

6 种诊断模式精准区分："人的问题"还是"策略的问题"？什么时候该管理介入，什么时候该调策略？这是大量实战中沉淀的模式识别。

🤖

System Prompt 设计

"虚拟策略分析师"：角色定义 + 数据上下文 + 输出格式约束。让 LLM 的输出稳定、可靠、可操作。

📚

Wiki 知识体系 (10+ 文档)

业务设计、报告规范、操作手册、数据字典……完整沉淀。避免因人员变动导致的工作中断，系统即是传承。

🔄

可复制性

新业务线接入只需配置数据源 + 复用三条线框架。已验证于 CL / BNPL / CC / Offline 四种场景。这不是工具复制，是思维框架的复制。

  • 时效性：归因分析从 2 小时缩短至 5 分钟

Part B

怎么构建的

从痛点到架构，从失败到成功，25+ 版本的完整技术日志

业务痛点

数据量 vs 人力

7 条业务线、5+ 逾期阶段、多维客群 — 数据组合爆炸式增长。

维度多 vs 视角窄

手工只看 1-2 个指标，客群结构变化、模型倒挂等信号被系统性忽略。

频率高 vs 产出慢

每日监控但人工数据处理 2-3 小时，等分析完已错过最佳干预窗口。

告诉你"为什么跌了"并量化"修复后整体能提升多少"。

def compute_smart_diagnostics(df_daily, df_process):
    window = build_monthly_matrix(df_daily, months=4)
    for group in window.groups:
        if consecutive_miss(group, threshold=3):
            flag_persistent_inefficiency(group)
    uplift = (module_avg - group_rate) * group_weight
    narrative = generate_diagnosis(group, pattern, uplift)
    return DiagnosisCard(severity, narrative, uplift)

# DPD 变化 = Mix Effect + Rate Effect

rate = Σ w_curr × (r_curr - r_prev)

构建实录 (4 Phase)

Phase 1: 从 0 到 1

自动化取数 + 基础报告

数据处理时间从 2-3h 缩短至自动化运行。

  • v1.0基础框架 — Excel→Python→HTML 最小闭环
  • v2.0-2.3全链路脚本 — 动态 SQL + 口径精细化
Phase 2: 深度归因

从"看到数"到"看懂数"

报告自动回答"为什么"。

  • v3.0-3.3MTD Lift + Safe Denominator + 动态新客识别
  • v3.5-3.9热力图 + 归因 + 效能排名
Phase 3: 体系重构

三段式布局 + 失败与重建

v4.3-4.6 四次回退。教训：增量开发必须基于稳定版本。

  • v4.0-4.2叙事重构 + 稳定基线
  • v4.3-4.6失败版本 — 改动范围过大
  • v4.8-4.18重建成功 + Shift-Share
Phase 4: 智能化

诊断 + 决策 + 追踪

首次具备"建议"能力。

  • v4.22Target Dashboard / Leaderboard / Efficiency / Action Items
  • v4.23Smart Diagnostics + Uplift 量化
  • v4.25数据质量校验模块 + 内存直出

下一步：动态案件路由 & 奖惩闭环

核心理念：游戏规则清晰 — 差组有惩罚、优秀组有奖励、问题明确、提升方向清晰。

🚨发现 & 惩罚
  • 连续 3 天下滑 → 减少新案分配
  • 收入随案量下降 — 压力信号
  • 明确告知差在哪、要改什么
🏆激励 & 奖励
  • 新案逐步分给优秀组
  • 回收增加 → 收入上升
  • 标杆经验可复制学习
🔄恢复 & 闭环
  • 差组修复后流量逐渐恢复
  • 动态路由 + 动态收入
  • 人定规则，系统执行验证
