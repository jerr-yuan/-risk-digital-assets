AI Agent System

智能数据巡检系统

基于 LLM 的自动化异动归因系统。每日自动分析回收率波动原因，从"人肉做数"到"AI 决策"。

5 min
归因分析时间（原需 2 小时）
90%
常规波动归因覆盖率
24x
效率提升倍数

1. 业务痛点

贷后策略团队每天需要花 1~2 小时分析前一天的回收率波动原因。这项工作：

  • 高度重复：90% 的波动原因可归类为短信通道故障、人员缺勤、案件结构变化等
  • 占用策略团队大量精力，导致"全员做数、无人决策"
  • 归因结论依赖个人经验，缺乏标准化

2. 系统架构

08:00

Data Fetcher

SQL 脚本自动拉取 T-1 运营数据

08:05

LLM Reasoning

大模型分析数据波动并归因

08:10

Auto Report

生成简报推送至飞书/钉钉

3. System Prompt 设计

Prompt 工程是本系统的核心。通过结构化的角色定义、数据上下文和输出格式约束，让 LLM 的输出稳定、可靠、可操作。

Role: You are a Senior Risk Analyst
      specializing in Debt Collection.

Context:
- Today's Date: {current_date}
- Target Metric: Daily Collection Rate (DCR)
- Actual DCR: {actual_dcr}
- Expected DCR: {expected_dcr}

Data Provided:
1. Hourly call volume data (Table A)
2. Case distribution by risk level (Table B)
3. SMS delivery success rates (Table C)

Task: Analyze the gap. Focus on:
1. Operational Efficiency
2. Risk Composition
3. Technical Issues

Output Format:
- Root Cause: [One sentence summary]
- Key Factors:
  - Factor 1: [Data evidence] -> [Impact]
  - Factor 2: ...
- Actionable Recommendation: [For tomorrow]

4. 技术栈

  • 数据层：SQL 定时任务从数仓拉取多维运营数据
  • 分析层：Python + LLM API（支持 GPT-4 / 千问）
  • 报告层：自动生成 HTML 巡检报告，支持飞书/钉钉推送
  • 迭代管理：完整的 Wiki 体系（业务设计、报告规范、操作手册、数据字典等 10+ 文档）

5. v4.23 智能诊断 (Smart Diagnostics)

系统最新版本引入了 6 种诊断模式识别引擎和 Uplift 量化能力，从"发现异常"进阶到"诊断原因 + 量化优先级"：

诊断模式 判定逻辑 建议动作
持续懈怠 (persistent_lazy)多月不达标 + 覆盖率/强度低加强管理考核
策略问题 (persistent_strategy)多月不达标 + 过程指标正常调整分案/策略
加速恶化 (accelerating_decline)达成率连续下滑紧急介入
新出现问题 (new_issue)近期新出现不达标关注观察
持续改善 (improving)达成率持续上升肯定 & 复制经验
稳定标杆 (stable_star)多月稳定达标标杆组织学习

Uplift 量化：如果不达标组提升至模块加权平均水平，对整体回收率的潜在提升（pp）。系统自动排序 Top 10 最值得修复的组，让管理决策有据可依。

6. 演进历程

v1 基础报表

手动 SQL + Excel → 自动化 HTML 日报。解决了"有没有"的问题。

v2 战略监控

三条线监控体系（风险/回收/过程）。解决了"看什么"的问题。

v3-v4 可视化交互

ECharts 图表、4 层下钻、Shift-Share 归因中心、Treemap 运营归因。解决了"怎么看"的问题。

v4.23 智能诊断

6 种模式识别 + Uplift 量化 + 自然语言叙事。解决了"看完之后怎么办"的问题。

7. 业务价值

  • 时效性：归因分析从 2 小时缩短至 5 分钟
  • 标准化：输出格式统一，覆盖 90% 常规波动
  • 决策赋能：团队从"做数"转向"决策"，策略迭代速度提升
  • 智能诊断：6 种模式识别区分"人的问题"vs"策略的问题"
  • 优先级量化：Uplift 排序告诉管理者先解决哪个组收益最大
  • 知识沉淀：完整 Wiki 体系（10+ 文档）+ System Prompt 将经验系统化
