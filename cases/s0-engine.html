S0 动态分案引擎

基于当天产能与案件风险等级的实时匹配系统。从传统固定规则到动态优化，最大化回收率。

1. 业务背景

传统分案采用固定规则（如按逾期天数平均分配），导致：

  • 高风险案件在低产能日（如周末）积压
  • 高能效催收员处理低价值案件，产能浪费
  • 无法根据当日实际情况动态调整

2. 解决方案：动态匹配算法

核心逻辑：基于当天的产能 (Capacity) 与案件风险等级 (Roll Rate Prediction) 进行实时匹配，最大化回收率。

def dynamic_assignment(cases, agents, strategy_params):
    """
    cases: 待分案池 (包含 risk_score, unpaid_amount)
    agents: 上线催收员 (包含 skill_score, capacity)
    strategy_params: 策略参数 (风险阈值, 负荷上限)
    """

    # Step 1: 案件分层 (Segmentation)
    high_risk = cases.filter(risk_score > params['threshold'])
    low_risk  = cases.filter(risk_score <= params['threshold'])

    # Step 2: 坐席分级 (Agent Ranking)
    senior = agents.filter(skill >= 'Senior').sort_by('capacity', desc=True)
    junior = agents.filter(skill < 'Senior')

    # Step 3: 优化匹配 — 高风险 → 高能力
    for agent in senior:
        capacity = agent.capacity * params['load_factor']
        assignments[agent.id] = high_risk.pop(top=capacity)

    # Step 4: 剩余 → 均匀分配
    distribute_evenly(low_risk, junior)

    return assignments

3. 关键创新

  • 动态灰度：全年灰度测试发现，高风险期应加大催收强度，低风险期应降低强度（非"一刀切"固定策略）
  • 人力预估：通过风险预测模型提前 T+7 天预估所需人力，联动外包/内催灵活调配
  • 周末/节假日自适应：自动降低分案量，避免积压

4. 业务价值

  • 回收率：全业务线回收率达历史最高水平，主力产品 CL 峰值 +3.0%，TT +5.3%
  • 人效：案量增长 80%，人力仅增长 25%
  • S0 优化：S0 阶段人力消耗较一年前大幅下降
