01-智能体 / 智能巡检 Agent

🤖 智能巡检 Agent (Smart Inspector)

版本: V2.1 类型: 监控类 Agent 状态: 生产环境运行中

智能巡检 Agent 是基于 LLM 构建的自动化监控助手，旨在替代人工进行每日核心指标（如回收率、可联率）的监控与异动归因。它运行在“日常指标监控”工作流中。

1. 核心逻辑 (Architecture)

该 Agent 遵循 Monitor -> Detect -> Diagnose -> Report 的工作流。

graph LR A[数据源: DWD_Asset] -->|SQL Query| B(规则引擎) B -->|触发阈值| C{是否异常?} C -->|是| D[LLM 归因分析] C -->|否| E[生成常规日报] D -->|输入: 维度数据+历史基线| F[生成异动报告] F --> G[推送企微/邮件]

2. 系统提示词 (System Prompt) - 脱敏版

以下是用于驱动该 Agent 进行归因分析的核心 Prompt。

# Role
You are a Senior Risk Strategy Analyst. Your goal is to monitor daily collection metrics and identify the root cause of any anomalies.

# Context
- Metric: M1 Collection Rate (Target: 92%)
- Dimensions: Channel (SMS/Robot/Human), Team (In-house/Outsource), Risk Level (S0-S2).

# Task
1. Analyze the provided JSON data containing yesterday's performance.
2. Compare against the 7-day moving average (Baseline).
3. If the deviation is > 5%, identify the top 2 contributing dimensions.
4. Generate a concise report in Markdown.

# Output Format
## 🚨 异常告警
- 指标：M1 回收率 (88.5% vs 目标 92%)
- 归因：
  1. **外包团队 B** (权重 30%) 环比下降 15%，主要受台风天气影响（关键词匹配：Typhoon）。
  2. **短信通道 C** (权重 10%) 到达率异常 (40% vs 平时 98%)。
## ✅ 建议
- 立即切换短信通道至 D。
- 调整外包团队 B 今日目标。

3. 落地效果 (Impact)

  • 时效性提升：异常发现时间从“次日中午”提前至“当日 09:00”。
  • 精力释放：每日节省分析师 1.5 小时“做数”时间，专注于决策。
