Methodology Framework

可复用的策略构建思维体系。从方法论到实践，每个框架都经过多个项目验证。

1

策略 0 → 1 方法论

沉淀的通用策略构建思维框架，用于指导新业务/新机制的从零搭建。已在 PK 机制、优惠券策略、新国别冷启动等多个项目中复用验证。 本质上是 "发现问题 → 定义规则 → 假设验证 → 算账投产" 的过程。

Phase 1

诊断与定调

痛点深挖 + 3 Keywords 战略目标拆解

Phase 2

机制设计

分层差异化 + 核心抓手 + 风控红线

Phase 3

验证与测算

核心假设 + A/B 实验 + ROI 盈亏测算

Phase 4

落地与执行

原子化 WBS + 甘特图 + PMO 管理

2

风控四类工作框架

明确工作分类，定义人与 Agent 的协作边界。策略团队的一切工作可归入四类，每类有清晰的人机分工。

# 工作类型 核心内容 频率 Agent 角色
1 日常指标监控 指标追踪、异常预警、报表 每日/每周 执行者
2 策略开发 新策略设计、底表构建 项目制 建造者
3 效果追踪迭代 回测验证、A/B 对比 持续 分析者
4 临时需求 取数、临时分析 随机 响应者

核心洞察

  • Agent 是"执行放大器"：人定义 1 份逻辑，Agent 执行 100 倍规模
  • 人的价值在"上游"：目标定义、逻辑拆解、最终决策
  • Agent 的边界是"主动性"：它只能等待指令，不会主动发起
  • 四类工作都需要人机协作：没有任何一类能完全交给 Agent
3

QBR 核心一页框架

季度/年度业务复盘的标准范式。确保从"做了什么"上升到"业务价值"与"未来延续性"。 结构：回顾三大突破 → 展望基于突破的次年目标。

2025 重点突破

数据底座 & 数据驱动管理

全场景口径标准化、维度打通、可复用；序时目标原子化。

2026 基于突破的目标

智能数据巡检

每日自动风险分析/异常归因，从"做数"到"决策"的跃迁。

2025 重点突破

S0 策略迭代

风险可控下 ROI 提升；高/低风险期动态强度与人力调配。

2026 基于突破的目标

动态资源配置 & PK 项目

先定义案件再匹配组织，用智能策略提升调配与评估效率。

4

指标预估管理

贷后策略最关键的能力之一：对核心指标（入催率、回收率、ROI 等）的准确预估。预估不仅服务于业务规划，更是策略投产决策和资源配置的基础。

Step 1

历史基线建立

基于 Vintage 矩阵和自然月序时数据，建立各阶段的历史基线

Step 2

驱动因子拆解

将指标拆解为可独立预测的因子：案量 × 回收率 × 费率

Step 3

场景化预估

Best / Base / Worst 三场景预估，量化不确定性范围

Step 4

动态校准

每月根据实际值 vs 预估值偏差校准模型参数，持续迭代

预估管理的价值

  • 资源配置：基于预估的人力规划和外包份额决策
  • 目标设定：自下而上的原子化目标 → 自然月/到期月序时目标
  • 风险预警：实际偏离预估 > 阈值时触发策略审查
  • 投产算账：新策略上线前的 ROI 预测 → 灰度测试验证
5

激励方案设计

催收绩效激励不仅仅是"多劳多得"——需要在回收目标、成本控制、风险红线三者之间找到最优解。 激励方案是策略落地的"最后一公里"，直接影响催收团队的行为模式。

设计维度 核心原则 实践案例
目标对齐 绩效目标 = 公司目标的原子化分解 将月度回收目标分解至组级/经办级日序时目标
差异化激励 不同阶段、不同资产类型需要不同激励力度 S0 轻触达激励 vs M2+ 高难度案件特殊激励
成本可控 激励总包不能超过回收增量的合理比例 ROI 红线 = 每增加 1 元激励至少带回 X 元回收
行为引导 激励什么行为，团队就产出什么结果 避免"只激励回收额"导致忽略过程质量
版本迭代 激励方案需要随业务变化持续迭代 OFW 绩效方案 V1 → V2 迭代：修正大额偏差
