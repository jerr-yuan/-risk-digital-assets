# 「时代意义」简历 / 作品集方案

参考 [Zara's AI learning library](https://zara.faces.site/ai)（[zara.faces.site/ai](https://zara.faces.site/ai)）的风格：**不只是 PDF 简历，而是一份可分享的「个人 AI + 风控」学习库 / 作品集站点**。

---

## 一、Zara 站点的特点（可借鉴）

Zara 的站点是用 [Faces](https://faces.app) 搭建的「AI 学习库」，结构大致包括：

- **精选资源**：自己看过并筛选的视频、播客、文章，附简短推荐语与分类（Fundamentals、Product、Prompting 等）。
- **人物与动态**：值得关注的 X/推特账号、Newsletter，体现你在关注谁、在学什么。
- **产品与工具**：自己在用的 AI 产品（如 NotebookLM、Granola 等），并说明为什么推荐。
- **Prompts / 可复用内容**：自己沉淀的提示词或模板（如 AI 词典、视频录制+实时提问、播客转书稿等），可直接展示或复用。

整体是「**个人品牌 + 学习轨迹 + 可复用产出**」的组合，比单页简历更有时代感、也更利于传播。

## 二、结合你的方向：AI 信贷风控 + 蚂蚁千问

可以做一个 **「袁鹏 · AI 信贷风控 / 风控科技」** 向的个人站点，结构可参考：

| 模块 | 内容建议 |
|------|----------|
| **关于我** | 简短介绍 + 当前简历核心信息（教育、度小满经历、核心项目），可链到 PDF 简历。 |
| **精选学习资源** | 大模型、风控、消费金融、蚂蚁/千问相关：视频、文章、报告、课程，附一两句推荐理由。 |
| **项目与产出** | 贷后企微、法催、坐席行为轨迹等项目的**脱敏版**说明；若有可展示的指标/看板/报告模板，可放截图或模板。 |
| **数据与建模** | 脱敏的指标定义、特征体系、策略/模型思路（不涉及真实数据），体现「数据建设 + 模型 + 策略」能力。 |

| **Prompts / 模板** | 你沉淀的、与风控/分析相关的提示词或报告模板（脱敏），方便访客复用或面试时展示。 |

这样既保留「传统简历」的求职功能，又有一份 **可链接、可迭代、带个人判断** 的「时代意义」作品集。

## 三、实现方式建议

### 1. 用 Faces 快速建站（最接近 Zara 风格）

- **Faces**：<https://faces.app>  
- 特点：低代码、模块化，适合做「个人主页 + 资源库 + 链接」类站点。  
- 做法：注册后选模板，按上面模块填内容；可先做「关于我 + 精选资源 + 项目摘要」，再逐步加 Prompts/模板。

### 2. 自建站点（Notion / 静态站 / 博客）

- **Notion**：用 Notion 页面做结构，再通过 Notion 自带的 Publish 或 NotionSite 等工具生成公开链接。  
- **静态站**：用 Next.js / Astro / Hugo 等 + 部署到 Vercel/Netlify，完全自定义版式和交互。  
- **博客/文档站**：用 MkDocs、Docusaurus、或当前仓库的 Markdown + GitHub Pages，把 `1.resume/` 和 `docs/` 的内容整理成「关于」「项目」「资源」「Prompts」等页面。

### 3. 本仓库内先做「可发布内容」

- 在 `1.resume/` 保持 PDF + Markdown 简历；在 `docs/` 或单独目录维护：
  - **精选资源列表**（风控、大模型、消费金融）：标题 + 链接 + 一两句推荐。
  - **项目说明**（脱敏）：每个项目一段话 + 关键结果/方法。
  - **Prompts / 模板**：与风控、AB 测试、报告相关的提示词或模板（纯文本/Markdown）。
- 之后可把上述内容迁移到 Faces / Notion / 静态站，形成正式站点。

## 四、与蚂蚁千问 / 风控岗位的衔接

- 站点中可明确写出 **「方向：蚂蚁千问等大模型在信贷风控中的应用」**，并引用公开信息（如蚂蚁消金多智能体风控、交互式智能风控、知识理解/推理等），体现你对行业方向的关注。

- 「Prompts / 模板」模块既能展示你的结构化思维，也能让 HR/面试官看到你可复用的产出习惯。
